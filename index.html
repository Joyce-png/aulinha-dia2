Como commitar e criar branches

  1. Criar repositório local, depois o arquivo via vscode
  2. Precisamos adicionar o .git no repositório: comando git init
  3. Para adicionar todas as alterações de todos os arquivos usa-se o comando "git add .", para adicionar as alterações de um arquivo específico, acrescenta o nome do arquivo. Exemplo: git add outro-arquivo.html
  4. Após adicionar (git add), pode ser geito o primeiro commit na branch principal, usando o comando "git commit" com a mensagem do commit. O comando fica: git commit -m "primeiro commit"
  5. Depois do primeiro commit, após realizadas as alterações necessárias no projeto, já pode-se transferir as alterações para o repositório remoto (GitHub). Lembrando que antes disso é necessário criar o repositório remoto no GitHub. Após a criação do repositório remoto, o próprio GitHub apresenta os próximos comandos a serem usados.
  6. Copiar e colar os comandos no terminal do vscode: git remote add origin https://github.com/repositorioexemplo.git git branch -M main git push -u origin main - a partir desse ponto as alterações posteriores seguem a mesma lógica para atualização sempre que feitas: adicionar (git add), commitar (git commit) e enviar para o servidor (git push).

Criando as ramificações

  1. Para criar a partir da main, a localização deve ser a main, usando o comando "git checkout -b" com o nome da nova branch entre aspas: git checkout -b "nova-branch-p2"
  2. Após as alterações na nova branch, segue os mesmos passos anteriores: adicionar (git add), commitar (git commit) e enviar para o servidor (git push).
  3. Para criar outra ramificação a partir da segunda, a localização deve estar na segunda e assim por diante

Outros comandos necessários no trabalho:

  1. Atualizar os dados do repositório remoto para local: usa "git pull"
  2. Clonar repositórios do git hub: vai no repositório online e no botão verde de "code" tem o link do repositório, copia esse link, cola no terminal dentro do vscode na linha de comando "git clone"
